# Feature: Budget Tracking

## Status
draft

## Goal
Enable users to define monthly budgets by category and track spending against those budgets, with rollover support and status alerts.

## Background
- [`moneybin-mcp.md`](moneybin-mcp.md) -- the partial budget mutation adapter
  and bespoke budget-status read are de-registered while this spec is draft.
  The old read synthesized budget-vs-actual from `BudgetService`, violating the
  convention that entries in the `reports` catalog read a `reports.*` view. The
  complete lifecycle mutation surface returns through normal tool admission,
  while report entries do not consume tool slots, once this spec ships the
  required models and operations (gated on M3C). `BudgetService` and the CLI
  remain available as implementation foundations.
- [`categorization-overview.md`](categorization-overview.md) -- Prerequisite for meaningful budgets (supersedes archived `transaction-categorization` spec)
- App schema: `src/moneybin/sql/schema/app_schema.sql`

## Requirements

1. Create monthly budgets by category with dollar amounts.
2. Track spending vs budget with status indicators (OK, WARNING at 90%, OVER).
3. Support budget periods with start/end months.
4. Support rollover: unspent budget carries forward to next month.
5. Provide historical budget performance (trend over months).
6. Support budget templates (copy last month's budgets to new month).
7. Calculate total budgeted vs total spent across all categories.


### New/modified tables

```sql
-- Budget rollover tracking
CREATE TABLE IF NOT EXISTS app.budget_rollovers (
    rollover_id VARCHAR PRIMARY KEY,
    budget_id VARCHAR NOT NULL REFERENCES app.budgets(budget_id),
    month VARCHAR NOT NULL,          -- YYYY-MM
    rollover_amount DECIMAL(18, 2),  -- positive = unspent, negative = overspent
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### Key decisions

- **Monthly granularity**: Budgets are always monthly. Weekly/quarterly can be derived.
- **Rollover is opt-in**: Default behavior is no rollover (each month starts fresh). Enable per-budget.
- **Budget status thresholds**: OK (< 90%), WARNING (90-100%), OVER (> 100%). Configurable per budget.
- **No automatic budget creation**: User explicitly creates budgets. AI can suggest based on spending patterns.
- **Budget read surface is a report, not a bespoke tool**: budget status and
  history are built as `reports.*` SQLMesh views plus `@report` runners and
  exposed through the single `reports(report_id=...)` catalog. `BudgetService`
  supplies mutation logic only; it does not synthesize report results.

### Planned mutation capabilities

- List active budgets.
- Create, update, disable, or delete a budget.
- Copy budgets from one month to another.
- Calculate and apply rollover for a month.

Future MCP capabilities remain unnamed until admission through the bounded
registry.
